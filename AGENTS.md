# AGENTS.md — Правила работы с проектом

## Общие принципы

1. **Интерфейс и сообщения** — весь пользовательский интерфейс и все сообщения бота должны быть на русском языке.

2. **Секреты** — токены, пароли и другие секреты хранятся **только** в переменных окружения (`.env` для локальной разработки, Variables на Railway). Никогда не добавлять секреты в код.

3. **Бизнес-логика** — бизнес-логика не размещается внутри Telegram-обработчиков. Обработчики должны быть тонкими и вызывать сервисы.

4. **Каталог и цены** — каталог товаров (`data/products.json`) и цены редактируются без изменения обработчиков бота.

5. **База данных** — проект временно работает только с SQLite.

## Перед коммитом

1. Запустите тесты: `pytest -v`
2. Исправьте все падающие тесты
3. Проверьте, что в коде нет секретов (BOT_TOKEN, пароли)
4. Проверьте импорты

## Railway

- **Не сломать Railway-деплой** — любые изменения должны сохранять работоспособность на Railway
- Бот работает как **worker-процесс** через long polling, не нужен HTTP-сервер
- SQLite-файл внутри Railway-контейнера может быть потерян при новом деплое

## Стиль кода

- Минимум комментариев в коде (только там, где логика неочевидна)
- Следуйте существующим naming conventions
- Используйте type hints
- Импорты: стандартная библиотека → сторонние → локальные

## Расширение функциональности

1. **Новый товар** → добавить в `data/products.json`
2. **Новый этап заказа** → добавить состояние в FSM и обработчик
3. **Новая страница/раздел** → добавить handler и keyboard
4. **Новая таблица в БД** → добавить модель в `models.py`

## Тестирование

- Тесты не должны обращаться к реальному Telegram API
- Использовать отдельную тестовую базу (in-memory SQLite)
- Покрывать: загрузку каталога, расчёт цен, генерацию номеров заказов, работу с БД

## Примеры изменений

### Добавить новый товар

```json
{
  "id": "new_product",
  "name": "Новый товар",
  "short_description": "Краткое описание",
  "full_description": "Полное описание",
  "dimensions": [],
  "materials": [],
  "available_colors": ["красный", "синий"],
  "image": null,
  "product_price": null,
  "service_fee": null,
  "delivery_price": null,
  "delivery_period": "уточняется",
  "is_custom": false,
  "is_active": true
}
```

### Изменить цены

Отредактировать `product_price`, `service_fee`, `delivery_price` в `data/products.json`. Если цена неизвестна — `null`.

### Добавить администратора

Добавить Telegram ID через запятую в `ADMIN_TELEGRAM_IDS`.
